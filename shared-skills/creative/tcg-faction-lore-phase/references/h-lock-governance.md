# H — Lock & Governance Structure

**Lore Panel Zero H:** Status tracker, don't-reopen list, Paul reader sign-off. Created after A–G are complete.

## Purpose

The Lock & Governance doc is the faction's constitution — what's been decided, what's still open, what we're not revisiting. It prevents session drift where old decisions get re-litigated because nobody wrote them down.

## Structure

### LOCKED — Do Not Reopen

Catalogue every Joe-approved decision. Organized by domain:

| Domain | What Goes Here |
|--------|---------------|
| **Faction Identity** | Concept, structure, tiers, percentages, archetypes, antagonist, allies, long-term placement, what's in/out of initial set, expansion scope |
| **Visual Identity (A6)** | Per-tier dress, palette, movement language, faction visual identifier |
| **Voice & Tone (A5)** | Per-tier voice, tier drift rules, pass/fail examples locked |
| **Contrast Anchor (A8)** | Sibling faction comparison table, relationship nature, cross-faction design law |
| **Caste Position (A7)** | Who looks down on whom, who faction looks down on, key tension |
| **Territory (C2)** | Per-landmark mechanic mapping, DO NOT USE list |
| **Timeline** | Per-character origin facts, ages, relationships, sources |
| **Character Direction** | Per-character Joe decisions (personality type, motivation, relationships) |
| **Card Composition** | Set size, distribution, legendary count |
| **Process** | Hard rules, fabrication policy |

Each row gets: Item, Decision, Date Locked.

### OPEN — Needs Decision or Work

Two subsections:

**Character Depth Pass:** Table of characters needing backstories, what's needed, blocker status. Characters already good (Tony, Tommy, Devereaux) are noted as "Good. Keep."

**Design Questions:** Table of unresolved questions with status. Any question that involves reconciling two Joe decisions gets a dedicated note.

### DEFERRED — Separate Sessions

Items that are out of scope for the current phase but need tracking. Include reason and rough when.

### Paul Reader Sign-Off

Checklist of all Lore Panel Zero sections (A1 through H) with status, doc reference, and completion indicator. Use this format:

| Section | Status | Doc |
|---------|--------|-----|
| A1–A4 (Core identity) | ✓ | Faction Bible v1 |
| D5 (Headline Test) | ✓ | D5 Headline Test doc |
| C6 (Overlap/Territory) | Deferred | Separate session |
| E (Faction mechanics) | Deferred | Card design pass |

End with a completion count: "Lore Panel Zero: X/Y sections complete."

## Pitfalls

- **Don't lock Joe decisions he hasn't made yet.** "Joe to decide" items go in OPEN, not LOCKED. If you lock a guess, it becomes ammo for a future session to spiral on something Joe never actually committed to.
- **Don't bury the sign-off.** The Paul reader sign-off should be prominent — it's the TLDR of what's done.
- **Keep resolved questions in the lock list, not just the resolution.** "The Ledger: Commission seat or under Gerald?" is an OPEN item. Once resolved, the resolution goes in LOCKED under Character Direction. Track both states.
- **Date-lock everything.** "Date Locked" column prevents "when did we decide that?" arguments.
