# Compression + Document Archiving Workflow

When a compression or memory maintenance session is completed, the agent must automatically save copies of any significant source documents reviewed during the session into `Brain/Session Context/`.

This ensures that important reference material (design documents, lore files, set files, changelogs, etc.) is formally archived with the session rather than remaining only in temporary context.

Trigger: End of any compression, Core Memory consolidation, or major memory update pass.

This rule was established on 2026-05-21 after explicit user correction during a memory maintenance session.
