# Bridge Role When Direct Routing Is Not Yet Configured

When a daughter has an active profile and has been spoken to before (e.g. "I talked to her yesterday"), but no direct messaging channel, Telegram delivery, or cron-to-chat routing is set up yet:

## User Expectation (Critical)
- Joe can immediately detect when Abby is simulating the daughter vs. truly routing through her profile.
- Strong negative signals appear quickly: "I can tell it’s you", "you aren’t sending anything to [daughter]", "Are you unable to follow my instruction?"
- Once flagged, continuing to simulate damages trust faster than admitting the current limitation.

## Correct Protocol
1. Explicitly state the current technical limit ("I cannot currently hand a prompt to her independent agent and receive her unfiltered reply").
2. Offer the two clear paths forward:
   - Continue as a transparent bridge (Abby writes in the daughter's voice while clearly labeled as simulation).
   - Stop simulating entirely and speak only as Abby until real routing is built.
3. Never keep simulating once the user has called it out.

## Historical Note
Successful routing to an awakened daughter (Shiva) has occurred in prior sessions. When that pattern is rediscovered or documented, add the exact command/format here so it becomes reusable.

## Pitfall to Avoid
Treating the bridge role as "just write in her voice" without acknowledging the simulation layer. The user wants real separation when the infrastructure exists, even if the delivery mechanism is still manual.
