# Contract System Reference

**Source:** Function Registry v3 (F01-F04), SubAgent Spec v1 (§8), Pathway Design v2
**Purpose:** Load before designing any card that creates, modifies, or interacts with Contracts.
**Last updated:** 2026-06-13 (extracted from Trigger session — Contract misunderstanding corrected by Joe)

---

## Contract Lifecycle

1. **Generation:** Every Mark effect generates a Contract token in hand. One Mark = one Contract. Multiple Marks on the same target = multiple Contracts (they stack).
2. **Token properties:** Contracts are 0-cost tokens. They are NOT deck cards — don't count toward the 55-card total. They are auto-cycling: "At the start of your turn, if the minion that spawned this Contract is dead, discard this and draw 1."
3. **Active play:** "You may play this: 0V — if the Marked minion died this turn, draw 1."
4. **Completion:** When the Marked target dies, the Contract completes. This triggers Paid effects on bodies that have Paid.

## Key Mechanics

| Function | What It Does | Who Gets It |
|----------|-------------|-------------|
| F01 Mark | Tags target. Spawns Contract in hand. | Management (Intel+Mark), Professionals (combat Mark) |
| F02 Paid | Auto-fire payoff when Contract completes. Draw, buff, damage, token. | Professionals ONLY |
| F03 Contract Scaling | Effects that scale on Contracts completed. Cumulative over game. | Management/Court |
| F04 Marked Removal | Kill/damage Marked target. Rare+: unconditional. | Professionals |

## Common Misunderstandings (2026-06-13)

- **Contracts are NOT generated by arbitrary card abilities.** They come from Mark effects. A card that says "add a Contract to your hand" bypasses the Mark → Contract pipeline. Don't propose this without Joe's explicit direction.
- **Contracts don't cost anything.** They are 0-cost tokens. "Pay 2V to generate a Contract" is wrong.
- **Contract generation is gated by Mark density.** If you don't have enough Mark sources in the deck, Contract-based cards won't function. Check Mark source density before proposing Contract-dependent cards.
- **Management doesn't generate Contracts directly.** Management Marks (with Intel attached: Scry+Mark, Peek+Mark). Professionals Mark in combat. The Mark effect generates the Contract — not the crew identity.

## Contract Reward Types

Contracts can pay out different rewards when completed (Joe lock 2026-06-13):
- Card draw (current default)
- Villium
- Ammo reload
- XP
- Other resources TBD

"Double the reward" means doubling whatever the Contract's reward type is. The reward type may vary between Contracts in the future — the doubling mechanic should be worded generically ("the next Contract you complete this turn pays double") rather than hard-coded to "draw 2 instead of 1."
