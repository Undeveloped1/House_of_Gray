# Five Crests Card Design Pipeline — Master Skill

Full original archived at `docs/Paul/workspace/five-crests-card-pipeline-archive.md`.

End-to-end process for producing a 55-card faction set from locked lore through autonomous sub-agent production.
Proven on Trigger v7 (A-) and Faceless v2 (A-). Compound trajectory: each faction starts higher.

## The Process (7 Phases)

### Phase 0: Identity Gate
Load `faction-identity-gate`. Score all 10 dimensions (1-5). Run 4 synthesis locks. **Gate:** 40+ total, 0 below 3, 0 drift locks. Below gate = fix identity.

### Phase 1: Spec Lock
Build: Function Registry (12-18), Synergy Web, Build Facts, Blind Spots, Crew CAN/CANNOT. Run `density_calc.py`. **Gate:** All docs cross-referenced, no contradictions.

### Phase 1.5: Build Facts Currency Check (MANDATORY)
Verify Build Facts date ≤ 1 day old, crew split matches latest Joe lock, curve matches current archetype. If stale: rewrite Build Facts or kill it if everything's duplicated in slot table/warmup/Spec. Re-derive density targets from fresh numbers. **Never build slot table from stale Build Facts.**

### Phase 2: Pathway Design
Load `pathway-design`. Build crew identity profiles, archetype turn maps (T1-T6), slot table (29 minion + 21 spell + 5 weapon). Calc density targets. **Gate:** Slot totals = card count. Crew % match targets.

### Phase 3: Spec Validation
Run validation directly if Paul hand-built the slot table (30-second counting task — never delegate counting). Spawn validator sub-agent only for autonomous/orchestrator-produced slots. **Gate:** 0 Critical findings.

### Pre-Flight Checklist (MANDATORY — before Phase 4)

| # | Check | If Missing |
|---|-------|-------------|
| PF1 | Phase 0: Scored identity gate | Run `faction-identity-gate` |
| PF2 | Phase 1: Locked specs | Lock Phase 1 |
| PF3 | Phase 2: Formal slot table | Run `pathway-design` |
| PF4 | Phase 3: Spec validation (0 Critical) | Run validator |

**Why:** Trigger v6/v7 scrapped — Phase 0 never run, Phase 2 never standalone delivery.

### Phase 4: Card Production

**Mode A: Collaborative Band-by-Band (PREFERRED)**
Joe+Paul design one V-band at a time. Per band: 2-3 proposals from slot table. Joe reacts. Lock or revise. **Session History Gate before designing any band** — verify via `session_search` it wasn't already completed. Warmup docs go stale; session history wins. After band lock: immediately update warmup + canonical Full Set in dropbox.

**Mode B: Autonomous Sub-Agent Production**
For speed or proven patterns. **Novelty Gate MANDATORY before spawning:** NO EXISTING NAMES directive, scrub card-name references from lore, post-production dedup check (>50% overlap = scrap). Spawn 3 parallel workers by V-band. Each gets full Spec + band instructions + format template.

### Phase 5: Automated Audit + Fixer
Run `pre_review_audit.py`. If FAIL → spawn ONE fixer for specific gaps. Never re-spawn all workers for small fixes. Max 2 fixer passes before escalate.

### Phase 6: Critic Review
Multi-perspective review (Tone, Design, Identity, Flavor, MTG Distinctiveness) or single critic. **Two-layer review minimum:** audit catches mechanical, critic catches ludo/flavor/identity. **Gate:** B+ or above.

### Phase 7: Delta Tracking & Compound
Record delta. Update COMPOUND_TRACKER. Meta-critic: "What broke? What would have prevented it?" Patch playbooks. Joe Surface Report.

## Critical Pitfalls

- **Contract system:** Before designing any card with "Contract" — load `references/contract-system-reference.md`. Contracts are 0-cost auto-cycling tokens generated by Mark, not arbitrary abilities.
- **TRACE NUMBERS TO SOURCE.** Never recite from memory. Cite document + date + assumptions. If you can't name the source, you're reciting — and it's probably stale.
- **Spec validation is counting, not reasoning.** Never `delegate_task` a 30-row checklist Paul can run in 30 seconds.
- **Spec must EXCLUDE, not just INCLUDE.** "NO SURNAMES" not "epithets/role-names."
- **Format drift is #1 friction.** Enforce templates. Verify with grep.
- **Workers must write FILES.** Explicitly: "Write output to [path]."
- **Two-layer review minimum.** Audit + critic. Neither alone is sufficient.
- **Sub-agents remix existing cards** when lore docs leak names. Faceless v2: 81% overlap → SCRAPPED.
- **Warmup docs go stale.** Session history is source of truth. When they disagree, session history wins.
- **Daily Handover can be stale after crash.** Canonical Full Set in dropbox is ultimate source of truth.
- **Skipping Phase 0 + Phase 2 = scrapped sets.** Pre-Flight Checklist is hard gate.
