# Faceless Upgraded Pipeline v2

Proven 11-phase card design process. Produced Faceless v2 set at 98.7% critic score.
Source: `/root/.hermes/docs/Paul/workspace/Faceless_Card_Design_Upgraded_v2_Paul.md`

## Phases

| Phase | Name | What It Does |
|-------|------|-------------|
| 1 | Functions Pass | One sentence per card. No numbers. Catch same-function redundancies BEFORE stats hide them. |
| 2 | Competitive Slotting | Cards fight for their slot. "Why wouldn't I always pick this?" Within-archetype, across-archetype, rank within cost tier. Bottom 20% are cut candidates. |
| 3 | Stress Test | Simulated combat vs aggro T1-T5. Does the faction stabilize? What's the win condition? |
| 4 | Build | Design cards INTO the curve from slotting analysis. Redesign cards with red flags. |
| 5 | Competitive Ranking | Rank every card in its archetype S/A/B/C/D-tier. Identify role players vs build-arounds. |
| 6 | Draw-at-Any-Turn | Score every card at T1, T5, T8. Is it dead early? Dead late? |
| 7 | One-Card Wonder Test | "Would a drafter open this in a pack and want to build around it?" All L and R must pass. |
| 8 | Cross-Splash Verification | Test all 6 cross-archetype splash combos at +1V tax. Does the tax break low-cost cards? |
| 9 | Rubric Scoring | Score refactored cards against Master Rubric Lanes M+N. |
| 10 | Patch Candidates | Generate playbook patches for every failure found. |
| 11 | Compound Tracker | Log scores, pass rates, patches. Track improvement across factions. |

## Key Innovations Over v1

v1: Design cards → check against rubric → iterate → patch.
v2: Functions pass → competitive slotting → stress test → design INTO the curve → ranking critic → cross-splash → draw-at-any-turn → pre-con teachability → iterate → patch.

The difference: v1 checks cards AFTER they exist. v2 designs cards IN COMPETITION with each other. Fewer iterations, higher hit rate, zero wasted slots.

## What It Catches That v1 Doesn't

1. **Same function at different costs** — Recruiter (3V) and Street Preacher (1V) both "summon a Lamb." Rubric checks keywords; slotting checks purpose.
2. **Linear upgrades of the same card** — Brother (3V) is just Hushed Witness (2V) with +1/+1 and +1 Purge. Passes rubric. Fails slotting.
3. **Canon contradictions in context** — Apostle had Blessed on a Sin card. Rubric would catch M1 violation, but slotting caught it WITH the changelog context.
